How to read a routing table

Sometimes we are thrown off guard by the huge number of lines in the routing table and troubleshooting becomes messy. All of us go through it, especially if you are not the one who designed the network and you are unclear about the topology that's in place and you are trying to troubleshoot the network issue. The key is not to read the entire table line by line. There is no point in comprehending the table entirely , instead concentrate on the things you are looking for.

In this article, we will go through some of the basics. There are many more parameters that need to be discussed, we can cover them in upcoming posts.

Looking for the right set of information in the routing table and understanding it step by step, makes the troubleshooting a cakewalk. Things that I like to check in the routing table first and foremost.

What are the sources of these routes?

The letter on the left most column of every route tells us the source of the route. Each letter is used to identify each protocol. This information is usually printed right there at the start of the output for Cisco routers #how ip route command. For other routers from different switch vendors, you will have to refer to the table below that helps you correlate the letter to the Routing protocol. This information also helps you to understand if the router is participating in dynamic routing protocols.

Is there a default route configured for the route?

It is a best practice to configure the default route for a router. Note down the default route for the router as in many cases, this can be a cause of the issue.

“*” next routing protocol identifier shows which among routes in the routing table is selected as the default router.

Longest prefix match

When we start troubleshooting, we usually start with the source and destination IP and try to perform a packet walk hop by hop. When we check if there is any route in the routing table that can help you to route the traffic to the destination IP, rather than matching all the lines, match the routes that have the longest prefix first. As the longest prefix would decide the subnet in which the IP is going to fall.

The destination IP addresses match all four entries in the routing table but the 192.168.16.0/27 has the longest prefix so it will be chosen. This is called the "longest prefix match" rule.

Administrative distance

When we have more than one route for the same network and the prefix matches as well, we decide the next hop based on the administrative distance. Administrative distance is the feature used by routers to select the best path when there are two or more different routes to the same destination from different routing protocols. Administrative distance defines the reliability of a routing protocol. Each routing protocol has its own priority and one routing protocol is better than the other for many reasons. Administrative distance helps us in evaluating the same. FYI, administrative distance of a route can be modified manually.
